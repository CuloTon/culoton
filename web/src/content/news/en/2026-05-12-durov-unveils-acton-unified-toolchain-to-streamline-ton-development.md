Pavel Durov has introduced Acton, a unified command-line toolchain for building smart contracts on The Open Network. The platform collapses what developers previously managed as separate tools into one integrated workflow covering project creation, code writing, testing, deployment, and on-chain verification.

According to Durov, the consolidation delivers a tenfold acceleration in development speed. Test execution runs roughly 50 times faster than before. The toolchain is built around Tolk, TON's native smart contract language, and integrates with mainstream code editors while offering built-in debugging and testing capabilities. Developers can simulate live blockchain conditions locally before pushing to mainnet.

A distinctive feature is Acton's native support for AI-assisted development. The platform includes documentation for AI coding agents and direct compatibility with tools like Codex and Claude, positioning it as infrastructure for autonomous, AI-driven development workflows on TON.

The launch coincides with Telegram's deepening control over TON's validator infrastructure. On May 4, Telegram staked 2.2 million tokens to secure the largest validator slot from the TON Foundation—a move framed within Durov's broader roadmap to strengthen the network. Recent upgrades have reduced transaction fees by six times and introduced Catchain 2.0, a consensus mechanism that pushed network finality below one second in April.

TON's transaction finality of 0.6 seconds positions it as the fastest Layer 1 blockchain by some measures, though Telegram's growing control of validator operations has drawn scrutiny from analysts flagging centralization risks. Acton's adoption by developers in the coming weeks will offer the first meaningful test of whether these infrastructure improvements can help TON compete with Solana and Ethereum in mini-app and commerce applications. Telegram's nearly one-billion-user base remains the singular distribution advantage rivals lack.
